# Site Map: Savheera Jewelry Landing Page

## Tujuan File

File ini menjelaskan **struktur halaman dan alur navigasi** untuk landing page.  
AI agent akan menggunakan panduan ini untuk membuat section `.js` dan menyusun layout secara konsisten.

## Folder Penyimpanan

src/app/bonus/landingpageTemplate/savheera/sections

## Struktur Halaman

1. **Hero Section**

   - File referensi: `SavheeraHero.js`
   - Tujuan: Menyambut pengunjung dengan tagline inspiratif dan visual utama

2. **Problem Section**

   - File referensi: `SavheeraProblem.js`
   - Tujuan: Menunjukkan kesulitan pengunjung memilih perhiasan untuk momen spesial

3. **Solution Section**

   - File referensi: `SavheeraSolution.js`
   - Tujuan: Menjelaskan bagaimana Savheera membantu menyelesaikan masalah

4. **Feature Section**

   - File referensi: `SavheeraFeature.js`
   - Tujuan: Menampilkan fitur unggulan, desain eksklusif, material premium

5. **Proof Section**

   - File referensi: `SavheeraProof.js`
   - Tujuan: Testimoni atau cerita pelanggan di momen spesial

6. **Pricing Section (Opsional)**

   - File referensi: `SavheeraPricing.js`
   - Tujuan: Paket atau opsi eksklusif, tetap soft selling

7. **CTA Section**

   - File referensi: `SavheeraCTA.js`
   - Tujuan: Mengajak pengunjung menjelajahi koleksi atau inspirasi momen

8. **Bonus Section**

   - File referensi: `SavheeraBonus.js`
   - Tujuan: Tips styling atau rekomendasi tambahan untuk momen spesial

9. **Footer Section**
   - File referensi: `SavheeraFooter.js`
   - Tujuan: Informasi kontak, sosial media, newsletter

## Catatan untuk AI Agent

- Gunakan file referensi yang tercantum untuk membuat section `.js` masing-masing.
- Semua section harus **mengikuti panduan brand** (warna, tipografi, spacing, layout).
- Urutan section harus **logis dan aspiratif**, tidak melakukan hard selling.
- Hero section sebagai **awal halaman**, kemudian lanjut ke Problem → Solution → Feature → Proof → Pricing → CTA → Bonus → Footer.
